结论摘要

本页只讨论完整 full8 benchmark 的方法结果。当前实验的主线是一个增量式 ablation：从 3D-only 出发，加入 2D+3D token early fusion，再扫描 token budget 和 3D:2D 配比，并用 answer-level fusion 做 late-fusion control。结果显示 early-fusion 能超过 16A objective，但尚未超过 RGB objective；最高 JudgeLM 和最高 Objective 仍不在同一方法上。

  • All previously partial phase2/hybrid candidates are now complete: phase2 audit is 40/40 scene-method pairs, and each completed method has 8 scenes / 1061 questions / 297 JudgeLM questions.
  • The strongest objective result is hybrid_T512_R30 at 78.01%, which is above 16A (74.61%) but still -1.70pp against the RGB target (79.71%).
  • The strongest JudgeLM result is farthest_3d at 8.192, compared with RGB 8.030; this shows that high open-answer quality and high objective accuracy are not yet achieved by the same method.
  • The main experimental signal is that token early-fusion improves objective accuracy over 3D-only and answer-level fusion, but the current fixed concatenation/interleaving designs are still short of the RGB objective target.

逐方法 Ablation 解释

这一节按一级主表里的每个方法/同分组逐条解释。每条只说明它自己的增量改动、实现方式、控制变量和结果解读；不额外拔高实验含义。

hybrid_T512_R30 +2

Objective 78.01% (-1.70pp vs RGB) · JudgeLM 7.963 (-0.067 vs RGB)

hybrid_T512_R30

What changed: Keep early fusion fixed but set total token budget to 512 and allocate 30% of tokens to 3D.

How it is implemented: The fused sequence uses a fixed 512-token budget; R30 means roughly 30% 3DGS tokens and 70% 2D rendered tokens before one VLM answer.

Controlled comparison: Compared with phase2 layout ablations, this isolates token budget and 3D:2D allocation while avoiding question-aware or scene-adaptive choices.

Interpretation: This row should be read as a budget-allocation ablation, not a new architecture; its value is showing which fixed 3D/2D ratio is most promising.

Family: Budget-ratio early-fusion · Variant: T512_R30

phase2_3d_first

What changed: Use the same early-fusion evidence as block concat but order 3D tokens before 2D tokens.

How it is implemented: The fused sequence places multiscale 3DGS tokens first, followed by the corresponding 2D rendered tokens, and performs a single VLM generation.

Controlled comparison: Compared with phase2_block, the evidence set is intended to stay the same; the ablation is whether modality order changes VLM use of context.

Interpretation: It ties block and hybrid_T512_R30 for best objective, so 3D-first ordering is not harmful and may be a clean default.

Family: Token early-fusion · Variant: 3D-first early fusion

phase2_block

What changed: Add 2D tokens to the 3D stream and concatenate the two modalities in blocks before answering.

How it is implemented: For each of the three scales, sampled 3DGS tokens and 2D rendered tokens are placed into a single sequence using block concatenation, then the VLM answers once from the fused evidence.

Controlled comparison: Compared with 3D-only methods, the main change is pre-answer access to both 2D appearance and 3D language tokens under a fixed budget.

Interpretation: This is tied for the best objective score, supporting early token fusion as the strongest current direction.

Family: Token early-fusion · Variant: Block early fusion

phase2_2d_first

Objective 76.83% (-2.88pp vs RGB) · JudgeLM 7.926 (-0.104 vs RGB)

phase2_2d_first

What changed: Use the same early-fusion evidence as 3d_first but place 2D tokens before 3D tokens.

How it is implemented: The VLM sees the rendered 2D multiscale tokens first and the 3DGS language tokens afterward in one fused sequence.

Controlled comparison: Compared with phase2_3d_first, only the modality order is changed; this isolates ordering sensitivity in the context sequence.

Interpretation: It drops below 3d_first/block, suggesting the current VLM benefits more when 3D evidence is introduced before 2D evidence.

Family: Token early-fusion · Variant: 2D-first early fusion

answer_fusion_3d_priority +1

Objective 76.05% (-3.66pp vs RGB) · JudgeLM 7.983 (-0.047 vs RGB)

answer_fusion_3d_priority

What changed: Move fusion after answering and use 3D as the default branch.

How it is implemented: The 2D and 3D branches answer separately; disagreements are resolved with a fixed 3D-priority rule.

Controlled comparison: Compared with 2D-priority answer fusion, the only change is the global branch priority; no question-aware routing is introduced.

Interpretation: It matches farthest_feature on the aggregate metrics, which suggests late fusion is not adding new joint reasoning beyond the stronger branch.

Family: Answer-level fusion · Variant: answer_fusion_3d_priority

farthest_feature

What changed: Keep the 3D-only setting but replace geometric diversity with language-feature diversity.

How it is implemented: The sampler chooses visible Gaussian tokens that are far apart in the 3DGS language-feature/codebook space before VLM answering.

Controlled comparison: Compared with farthest_3d, the token budget and one-shot VLM protocol are unchanged; only the 3D token selection criterion changes.

Interpretation: This is the best 3D-only objective variant, indicating that semantic diversity is more useful than pure geometry for objective questions.

Family: 3D-only token sampling · Variant: farthest_feature

phase2_token_interleave

What changed: Use the most fine-grained early-fusion layout by alternating individual 2D and 3D tokens.

How it is implemented: After sampling both modalities, tokens are interleaved at token granularity before one VLM answer is generated.

Controlled comparison: Compared with level_interleave, the change is finer modality alternation while preserving the same no-question-aware setup.

Interpretation: It is better than level_interleave but still below block/3d_first, suggesting too much alternation may disrupt useful modality structure.

phase2_level_interleave

Objective 75.13% (-4.58pp vs RGB) · JudgeLM 7.889 (-0.141 vs RGB)

phase2_level_interleave

What changed: Keep early fusion but interleave by scale level rather than by modality block.

How it is implemented: Small/Medium/Large 2D and 3D token groups are alternated so each scale contributes before moving to the next scale.

Controlled comparison: Compared with block/3d_first/2d_first, this changes the scale-layout of the same multiscale 2D+3D evidence.

Interpretation: It underperforms the best early-fusion layouts, so scale-level alternation alone is not the right organization.

Family: Token early-fusion · Variant: Scale-level interleaving

answer_fusion_2d_priority

What changed: Move fusion after answering and use 2D as the default branch.

How it is implemented: The 2D and 3D branches produce answers separately; if they disagree, a fixed 2D-priority rule selects the final answer.

Controlled comparison: Compared with token early-fusion, this tests whether late answer selection can replace joint token evidence fusion.

Interpretation: It mostly behaves like the 2D branch and does not recover the objective gain of token early-fusion.

opacity_topk

Objective 73.30% (-6.41pp vs RGB) · JudgeLM 7.811 (-0.219 vs RGB)

opacity_topk

What changed: Use visibility confidence as the 3D-only token selection rule.

How it is implemented: The sampler keeps the highest-opacity visible Gaussians and discards lower-opacity candidates before VLM answering.

Controlled comparison: Compared with diversity-based 3D sampling, this asks whether confidence/visibility alone is a sufficient proxy for semantic usefulness.

Interpretation: It is below feature-diverse 3D sampling, suggesting high-opacity Gaussians are not necessarily the most discriminative language tokens.

Family: 3D-only token sampling · Variant: opacity_topk

hybrid

Objective 72.91% (-6.81pp vs RGB) · JudgeLM 7.859 (-0.172 vs RGB)

hybrid

What changed: Mix geometric and feature-space criteria inside the 3D-only sampler.

How it is implemented: Part of the visible Gaussian budget is selected for 3D spatial coverage and part for language-feature diversity.

Controlled comparison: Compared with farthest_3d/farthest_feature, this tests whether combining two 3D diversity criteria is better than either alone.

Interpretation: The mixed 3D-only strategy does not dominate the single-criterion variants, so the bottleneck is not solved by simply mixing 3D sampling heuristics.

Family: 3D-only token sampling · Variant: hybrid

opacity_weighted

Objective 72.77% (-6.94pp vs RGB) · JudgeLM 8.024 (-0.007 vs RGB)

opacity_weighted

What changed: Soften the opacity-only selection rule while staying in the 3D-only setting.

How it is implemented: Visible Gaussians are sampled with opacity-weighted coverage instead of strict top-k selection.

Controlled comparison: This controls for the same 3D-only evidence source and changes only how opacity affects token selection.

Interpretation: It improves JudgeLM over opacity_topk but still does not close the objective gap to RGB.

Family: 3D-only token sampling · Variant: opacity_weighted

farthest_3d

Objective 68.46% (-11.26pp vs RGB) · JudgeLM 8.192 (+0.162 vs RGB)

farthest_3d

What changed: Use 3DGS geometry rather than 2D rendered features as the only evidence stream.

How it is implemented: Visible Gaussian language tokens are sampled by farthest-point selection in 3D coordinate space, then passed as one token sequence to the VLM.

Controlled comparison: Controls out all 2D token evidence and tests whether spatial coverage of the reconstructed 3D scene is enough.

Interpretation: This variant gives the strongest JudgeLM score, suggesting geometrically diverse 3D evidence helps open-ended landmark descriptions, but its objective accuracy is weak.

Family: 3D-only token sampling · Variant: farthest_3d
